# 關注點分離
* **`core.py` 關注的是資料邏輯（資料長什麼樣、怎麼算分數）**
* **`report.py` 關注的是呈現邏輯（報表長什麼樣、如何排版）**
> **`低耦合`：`core` 跟 `report` 互相獨立，改一個不會壞另一個。<br> `高內聚`：每個檔案都專心做好自己份內的事**

## 程式的核心
把『資料計算』獨立放在 `core.py`，把『報表輸出』獨立放在 `report.py`。主程式 `__main__.py` 只需要透過 `import` 把需要的功能『引入』進來，再照順序執行就好。這樣以後如果我們要新增報表格式，只要改 `report.py` 就可以了，完全不會動到核心的計算邏輯！
